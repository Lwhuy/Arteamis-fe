# P8.4 — Handoff (Work Package + Agent Brief) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Turn an accepted decision or belief into a governed unit of work — a `work_package` assigned to a human or an agent, carrying (for agent assignees) an `agent_brief` that records the objective, allowed context, budget, and approval gate. This is the **record-only handoff**: no agent runs in this phase, nothing here submits a background job — it captures what *would* be delegated so a human can review the brief before anything executes.

**Architecture:** Single-tenant, workspace-ready (Option B), continuing the pattern established in P8.2 (Promotion Bridge) and assumed-landed in P8.3 (Decision/Rule). New SurrealDB table `work_package` + graph edge `executes` (`work_package -> decision|belief`), migration 24. Domain model `WorkPackage` extends `ObjectModel` in the existing `open_notebook/domain/governance.py`. Service functions live in the existing `api/governance_service.py`; router endpoints in the existing `api/routers/governance.py` (already registered in `api/main.py` — no router-registration changes needed). Frontend adds work-package types/hooks to the existing `governance.ts`/`use-governance.ts`, a `CreateWorkPackageDialog` with an `AgentBrief` sub-form, a `WorkPackagesSection` sidebar list (Company scope) with inline status advancement, and a `CreateWorkPackageButton` "assign" affordance wired into `CompanyBrainSection` (beliefs) today — with an explicit note on wiring the same button into the decision list once P8.3's frontend lands.

**Tech Stack:** SurrealDB migrations, FastAPI, Python 3.12 (`uv`, pytest, ruff), surreal graph edges, Next.js 16 / React 19 / TanStack Query 5 / vitest, `@radix-ui/react-dialog` + `@radix-ui/react-checkbox` shadcn primitives.

- **i18n is test-enforced across all 14 locales** (`bn-IN`, `ca-ES`, `de-DE`, `en-US`, `es-ES`, `fr-FR`, `it-IT`, `ja-JP`, `pl-PL`, `pt-BR`, `ru-RU`, `tr-TR`, `zh-CN`, `zh-TW` under `frontend/src/lib/locales/`). `frontend/src/lib/locales/index.test.ts` enforces (a) key parity — every locale must have exactly the same key set as `en-US`, and (b) unused-key detection — every `en-US` leaf key must appear **literally** as a string somewhere in a source file. This means **no dynamically-built i18n keys** (e.g. `t(\`controlPlane.workPackage.status.${status}\`)`) — use a literal lookup map instead, so the key string physically exists in the file.
- **HTTP via `apiClient`, TanStack Query hooks in `frontend/src/lib/hooks/`.** `apiClient` is imported as a default export (`import apiClient from '@/lib/api/client'`). Mutations invalidate the relevant query key(s) and show a toast via `useToast()`; strings routed through `t()`.
- **Single-tenant / workspace-ready:** `work_package` carries `workspace: option<record<workspace>>` (nullable, `FLEXIBLE`), matching migrations 20–23.
- **Promotion is still the only private→company bridge** (P8.2); this plan doesn't touch that boundary. `work_package` is a Company-scope construct that consumes already-promoted `decision`/`belief` records.
- **Audit every transition.** `create_work_package` writes `audit_event(action='work_package.created')`; `update_work_package_status` writes `audit_event(action='work_package.status_changed', meta={'from':..., 'to':...})`.
- **Agent execution is explicitly OUT of scope.** `create_work_package`/`agent_brief` only *record* the brief (objective, allowed context, budget, approval gate). Nothing in this plan calls `submit_command()`, imports from `commands/`, or provisions an LLM. The `approval_gate` field is a recorded intent, not an enforced runtime check — there is no runtime to check it against yet.
- Backend: `uv run pytest tests/`, `ruff check . --fix`. Frontend (`frontend/`): `npm run test`, `npm run lint`, `npm run build`.

```surql
-- open_notebook/database/migrations/24.surrealql
-- Migration 24: work_package table + executes edge (Handoff, P8.4).
-- A work_package turns an accepted decision/belief into a governed unit of
-- work: a human or an agent is assigned, and — for agent assignees — an
-- agent_brief records objective/allowed_context/budget/approval_gate so the
-- brief can be reviewed before any agent actually runs. Agent execution
-- itself is out of scope for this migration/plan: work_package only
-- *records* the handoff. Mirrors migration 20-23's workspace-ready idiom
-- (option<record<workspace>>, nullable, FLEXIBLE).

DEFINE TABLE work_package SCHEMAFULL;
DEFINE FIELD IF NOT EXISTS workspace     ON TABLE work_package FLEXIBLE TYPE option<record<workspace>>;
DEFINE FIELD IF NOT EXISTS title         ON TABLE work_package TYPE string;
DEFINE FIELD IF NOT EXISTS assignee_kind ON TABLE work_package TYPE string DEFAULT 'human' ASSERT $value IN ['human','agent'];
DEFINE FIELD IF NOT EXISTS assignee      ON TABLE work_package TYPE option<string>;
DEFINE FIELD IF NOT EXISTS status        ON TABLE work_package TYPE string DEFAULT 'open' ASSERT $value IN ['open','running','done'];
DEFINE FIELD IF NOT EXISTS agent_brief   ON TABLE work_package FLEXIBLE TYPE option<object>;
DEFINE FIELD IF NOT EXISTS created       ON work_package DEFAULT time::now() VALUE $before OR time::now();
DEFINE FIELD IF NOT EXISTS updated       ON work_package DEFAULT time::now() VALUE time::now();

-- edge: a work_package executes a decision or a belief.
DEFINE TABLE IF NOT EXISTS executes
TYPE RELATION
FROM work_package TO decision|belief;
```

```python
ASSIGNEE_KINDS = ["human", "agent"]
WORK_PACKAGE_STATUSES = ["open", "running", "done"]


# Add after the AuditEvent class:
class WorkPackage(ObjectModel):
    table_name: ClassVar[str] = "work_package"
    title: str
    assignee_kind: str = "human"
    # NOTE: plain string, not record<user> — a work package can be assigned
    # to an "agent" that has no user record. See the CRITICAL LESSON note in
    # this file's docstring-equivalent (Global Constraints of the P8.4 plan)
    # and tests/test_governance_record_links.py::
    # test_work_package_assignee_is_left_as_plain_string for why this does
    # NOT get a _prepare_save_data override like Proposal.author does.
    assignee: Optional[str] = None
    status: str = "open"
    agent_brief: Optional[Dict[str, Any]] = None

    @field_validator("assignee_kind")
    @classmethod
    def _assignee_kind(cls, v: str) -> str:
        if v not in ASSIGNEE_KINDS:
            raise ValueError(f"invalid assignee_kind {v}")
        return v

    @field_validator("status")
    @classmethod
    def _status(cls, v: str) -> str:
        if v not in WORK_PACKAGE_STATUSES:
            raise ValueError(f"invalid status {v}")
        return v
```

```python
async def create_work_package(
    actor: str,
    *,
    title: str,
    assignee_kind: str,
    assignee: Optional[str],
    agent_brief: Optional[dict[str, Any]],
    executes_ids: list[str],
) -> WorkPackage:
    """Turn an accepted decision/belief into a governed unit of work.

    Links work_package->executes->{decision|belief} for every id in
    `executes_ids`. Agent execution itself is out of scope here — this only
    records the brief; nothing in this function runs an agent or submits a
    background command.
    """
    work_package = WorkPackage(
        title=title,
        assignee_kind=assignee_kind,
        assignee=assignee,
        agent_brief=agent_brief,
        status="open",
    )
    await work_package.save()
    for target_id in executes_ids:
        await repo_relate(work_package.id, "executes", target_id, {})
    await _audit(
        actor,
        "work_package.created",
        work_package.id,
        {"assignee_kind": assignee_kind, "executes": executes_ids},
    )
    return work_package


async def list_work_packages(*, status: Optional[str] = None) -> list[WorkPackage]:
    """List work packages, optionally filtered by status.

    `WorkPackage.get_all()` only supports an ORDER BY clause (no WHERE), so
    the status filter is applied in Python — matching `list_proposals`.
    """
    work_packages = await WorkPackage.get_all()
    if status is not None:
        work_packages = [w for w in work_packages if w.status == status]
    return work_packages


async def get_work_package(work_package_id: str) -> WorkPackage:
    return await WorkPackage.get(work_package_id)


async def update_work_package_status(
    actor: str, work_package_id: str, status: str
) -> WorkPackage:
    """Transition a work package's status, auditing the before/after.

    Raises:
        ValueError: if `status` is not one of WORK_PACKAGE_STATUSES.
    """
    if status not in WORK_PACKAGE_STATUSES:
        raise ValueError(f"invalid status {status}")
    work_package = await WorkPackage.get(work_package_id)
    previous_status = work_package.status
    work_package.status = status
    await work_package.save()
    await _audit(
        actor,
        "work_package.status_changed",
        work_package.id,
        {"from": previous_status, "to": status},
    )
    return work_package
```

> `update_work_package_status` validates `status` against `WORK_PACKAGE_STATUSES` explicitly (rather than relying on Pydantic field-level validation firing on assignment) because `ObjectModel` doesn't set `validate_assignment=True` — `work_package.status = status` would silently accept any string otherwise. This mirrors why `accept_proposal`/`request_changes` guard `proposal.status` explicitly before mutating it.

```tsx
// frontend/src/components/control-plane/WorkPackagesSection.tsx
'use client';
import { useWorkPackages, useUpdateWorkPackageStatus } from '@/lib/hooks/use-governance';
import { useTranslation } from '@/lib/hooks/use-translation';
import { cn } from '@/lib/utils';
import type { WorkPackage } from '@/lib/api/governance';

const NEXT_STATUS: Record<WorkPackage['status'], WorkPackage['status'] | null> = {
  open: 'running',
  running: 'done',
  done: null,
};

const STATUS_LABEL_KEY: Record<WorkPackage['status'], string> = {
  open: 'controlPlane.workPackage.status.open',
  running: 'controlPlane.workPackage.status.running',
  done: 'controlPlane.workPackage.status.done',
};

export function WorkPackagesSection() {
  const { t } = useTranslation();
  const { data, isLoading } = useWorkPackages();
  const updateStatus = useUpdateWorkPackageStatus();
  const items = data ?? [];

  if (isLoading) return <div className="text-xs text-muted-foreground">{t('common.loading')}</div>;
  if (items.length === 0)
    return (
      <div className="rounded-lg border border-dashed border-border p-3 text-center text-xs text-muted-foreground">
        {t('controlPlane.sidebar.workPackagesEmpty')}
      </div>
    );

  return (
    <div className="flex flex-col gap-2">
      {items.map((wp) => {
        const next = NEXT_STATUS[wp.status];
        return (
          <div key={wp.id} className="rounded-xl border border-border bg-card p-3">
            <div className="text-xs font-bold text-foreground">{wp.title}</div>
            <div className="mt-1 flex items-center gap-2 text-[10px] text-muted-foreground">
              <span
                className={cn(
                  'rounded-full px-2 py-0.5 font-bold',
                  wp.assignee_kind === 'agent' ? 'bg-primary/15 text-primary' : 'bg-muted text-muted-foreground',
                )}
              >
                {t(
                  wp.assignee_kind === 'agent'
                    ? 'controlPlane.workPackage.assigneeKindAgent'
                    : 'controlPlane.workPackage.assigneeKindHuman',
                )}
              </span>
              <span>{t(STATUS_LABEL_KEY[wp.status])}</span>
            </div>
            {next && (
              <button
                type="button"
                onClick={() => updateStatus.mutate({ id: wp.id, status: next })}
                disabled={updateStatus.isPending}
                className="mt-2 rounded-md px-2.5 py-1 text-xs font-semibold text-primary hover:underline"
              >
                {t(
                  next === 'running'
                    ? 'controlPlane.workPackage.startAction'
                    : 'controlPlane.workPackage.completeAction',
                )}
              </button>
            )}
          </div>
        );
      })}
    </div>
  );
}
```
