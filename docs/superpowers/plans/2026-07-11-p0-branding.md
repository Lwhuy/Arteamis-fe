# P0 — Branding (Arteamis favicon + color palette) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Rebrand Arteamis-fe from Open Notebook defaults to the Arteamis identity by installing the Arteamis favicon/PWA-icon set and swapping the values (not names) of the oklch shadcn design tokens in `globals.css`, touching no application logic.

**Architecture:** Presentation-only phase. Two source files change (`frontend/src/app/layout.tsx`, `frontend/src/app/globals.css`), one stale asset is removed (`frontend/src/app/favicon.ico`), and seven brand assets are copied into `frontend/public/favicon/`. Tailwind v4 reads the `--*` CSS variables in `globals.css`; because only the literal `oklch(...)` values change and every token name is preserved, all existing components re-theme automatically with zero component edits.

**Tech Stack:** Next.js 16 (App Router, `next/font/google`), Tailwind v4 (oklch shadcn tokens, `.dark` class via next-themes), TypeScript, ESLint, `npm run build` as the compile gate.

**Spec:** docs/superpowers/specs/2026-07-11-p0-branding-design.md
**Depends on:** nothing (P0 is fully independent) · **Branch:** feat/auth-multitenancy

## Global Constraints
- This phase edits exactly two source files (`layout.tsx`, `globals.css`), removes one stale file (`src/app/favicon.ico`), and copies asset files into `public/favicon/`. Touch NO backend, migration, i18n, component, or hook file.
- Token **names** are frozen: the `@theme inline` block (lines 7-45) and `@layer base` block (lines 116-201) of `globals.css` are NOT edited. Only the `oklch(...)` literals inside `:root {}` (lines 47-80) and `.dark {}` (lines 82-114) change. `--radius: 0.65rem` is unchanged.
- No new i18n keys: `title`/`description` in `layout.tsx` are hardcoded literals today (not `t()` calls), so no locale files are touched.
- `title`/`description` stay hardcoded (do NOT route them through i18n in this phase).
- Verification gate is the build + lint (there is no unit test for CSS values); plus two concrete invariant guards (below) that must survive the swap.
- **Two design-system invariants that MUST hold after the swap** (spec §"Two design-system rules"):
  1. `--primary-foreground` is ink `oklch(0.269 0 90)` in BOTH `:root` and `.dark` (label on coral is ink, not white — coral is only 3.12:1 on white but AA on ink).
  2. Coral `oklch(0.672 0.131 38.8)` is used ONLY as a fill (`--primary`, `--ring`, `--chart-1`, `--sidebar-primary`, `--sidebar-ring`). No `*-foreground` / text token is ever set to raw coral.
- Frontend commands run inside `frontend/`: `npm run lint` · `npm run build`.
